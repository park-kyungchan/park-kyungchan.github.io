# 편집 가이드 — 김현정 수학학원 소개 페이지

## 파일 구조

이제 사이트가 역할별로 나뉘어 있습니다.

| 파일 | 내용 | 보통 여기서 편집? |
|---|---|---|
| **content.json** | 화면에 보이는 모든 글자(~108개) | ✅ **텍스트는 여기서** |
| `kc_intro.html` | 구조(마크업) — `build.py`가 텍스트를 박아 넣는 대상 | 구조 바꿀 때만 |
| `styles.css` | 색·간격·폰트 등 스타일 | 디자인 바꿀 때 |
| `app.js` | 스크롤·애니메이션·지식그래프 동작 | 동작 바꿀 때 |
| `kg-data.js` | 지식그래프 데이터(개념·연결·학생상태) | 그래프 내용 바꿀 때 |
| `build.py` | content.json → kc_intro.html 반영 스크립트 | 실행만 |

> ⚠️ **push 할 땐 5개 파일을 모두 함께** 올리세요. 하나라도 빠지면 사이트가 깨집니다.

---

## 1. 텍스트 고치기 (가장 자주 하는 일)

1. **`content.json`** 을 편집기로 엽니다.
2. 고칠 문구를 찾습니다. 키가 `섹션_역할_번호` 라서 찾기 쉽습니다:
   - `sec1_title_1` = 첫 화면 큰 제목, `sec1_body_1` = 그 아래 문장
   - `_title`=제목 · `_body`=본문 · `_eyebrow`=작은 라벨(APPROACH 등) · `_cap`=원형 캡션(학기/주…) · `_stepnum`=STEP 01 · `_steplabel`=STEP 가운데 글자 · `_chtitle`/`_chdesc`=연락처 · `_tel`=전화번호
3. 따옴표 **안의 글자만** 고칩니다. 예:
   ```json
   "sec1_title_1": "왜 부터<br>이해시키는<br>수학 수업",
   ```
4. 저장 → 터미널에서 **`python build.py`** 실행 → 브라우저 새로고침.

> `build.py` 는 content.json 의 글자를 kc_intro.html 에 그대로 박아 넣습니다(런타임에 따로 불러오지 않아 검색·공유 미리보기에 그대로 잡힙니다). **텍스트의 정답은 content.json** 이니, kc_intro.html 을 직접 고치면 다음 build 때 덮어써집니다.

---

## 2. 줄바꿈 규칙 — `<br>` 하나로 통일

값 안에서 줄을 바꾸려면 `<br>` 를 넣습니다. (소스의 Enter·공백은 화면에 영향 없음 — 오직 `<br>` 만 줄을 바꿈)

| 쓰는 법 | 동작 |
|---|---|
| `<br>` | 항상 줄바꿈 (PC·모바일 모두) |
| `<br class="dbr">` | **PC에서만** 줄바꿈 (모바일에선 한 줄로 이어짐) |

```json
"sec6_title_1": "수학에는<br class=\"dbr\">지름길이<br class=\"dbr\">없습니다"
```
JSON 안에서 `class="dbr"` 의 따옴표는 `\"` 로 써야 합니다(위 예시처럼).

---

## 3. 실시간 미리보기

- 간단: `kc_intro.html` 더블클릭 → 수정/빌드 후 F5.
- 자동 새로고침: VS Code `Live Server`, 또는 `python -m http.server` 후 `http://localhost:8000/kc_intro.html`.
- 모바일 확인: 개발자도구(F12) → 기기 툴바(Ctrl+Shift+M) → 폭 640px 이하. 이때 `dbr` 줄바꿈이 사라집니다.

---

## 4. 배포 (사이트 반영)

```
python build.py
git add kc_intro.html content.json styles.css app.js kg-data.js
git commit -m "내용 수정"
git push
```
