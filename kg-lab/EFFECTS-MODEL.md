# KG 3D — 애니메이션 효과 모델 (구현 합의 문서)

> 대상 파일: `kg-lab-3d.html` (단일 리드 백엔드 — 3d-force-graph / Three.js).
> 2D 파일럿(Canvas/PixiJS)은 폐기됨. 모든 효과는 **`__KGGRAPH` / `__KGSTATE` 계약만** 의존하며
> 노드 id를 하드코딩하지 않으므로, SSoT 온톨로지나 학생별 KG를 교체해도 코드 수정 없이 동작합니다.

## 0. 공통 구조 — 효과 레지스트리
- 패널 **"효과 · 라이브 비교"** 섹션에 효과 패밀리별 변형 버튼 한 줄씩.
- `registerFX(name, label, options, initial)` 로 등록 → 변형은 라디오. `window.__FX[name]` 로 현재 변형 조회.
- **프리셋**: `추천 연출`(idle=은은·flow=강화·click=펄스·camera=회전) / `전부 끔`.
- **focus-set 파생은 단일 지점** `isFocusNode(id)` (schema의 focus 규칙: `focus` 배열 우선, 없으면 current/learning/weak 파생).

## 1. 앰비언트 모션 (idle) — "살아있는 신경망"
| 변형 | 동작 |
|---|---|
| 끔 | 정적 |
| 호흡 | 전 노드 미세 스케일 맥동(±3.5%), 노드별 위상차 |
| 반짝임 | emissive 미세 셰이머(별빛) |
| 은은 | 호흡 + 반짝임 |
- 모든 뷰(상태/구조)에서 동작. 상태별 맥동과 **곱연산 합성**.
- (참고) 노드 위치 "떠다님"은 링크 분리 위험 때문에 보류 — 전체 씬 부유는 카메라 연출에서 다룸.

## 2. 인트로 / 리빌 (intro) — 로드 시 조립
| 변형 | 동작 |
|---|---|
| 끔 | 즉시 표시 |
| 조립 | 레이어 순(DATA→LOGIC→ACTION→PROBLEM) 스태거 팝인 |
| 점화 | **focus-set에서 BFS 웨이브**가 바깥으로 확산 (이번 주 영역에서 신호가 퍼짐) |
- 변형 버튼 재클릭 = 리플레이. 종료 시 `revealAll()`로 전부 복원(스케일1·불투명0.95) 보장.

## 3. 흐름 파티클 (flow) — 이해 전파
| 변형 | 동작 |
|---|---|
| 끔 | 파티클 없음 |
| 기본 | 충전 노드(mastered/current/learning)에서 방향대로 방출 (DIRECTION-MODEL §A) |
| 강화 | **focus 노드에 닿는 엣지만** 발광 + 파티클 크게/많게, 비-focus 엣지 **완전 정지** |
| 강렬 | 강화 + 더 굵은 라인 글로우·더 많은 파티클 |
- 강화/강렬은 "이번 주 활동 영역"만 살아있게 하여 한눈에 도드라지게 함(의도 확정 사항).
- 시간 범위(이번달/주/차시) 개념은 추후 — 지금은 단일 focus-set.

## 4. 클릭 피드백 (click)
| 변형 | 동작 |
|---|---|
| 기본 | 카메라 플라이 + 상세 패널 (기존 T6) |
| 펄스 | 클릭 노드가 팝(스케일·발광) |
| 체인 | 펄스가 연결을 따라 BFS(≤3단계) 확산 — 신호 점화 |
- 호버 하이라이트(이웃 강조·비이웃 dim)는 전 변형 공통 유지.

## 5. 카메라 연출 (camera)
| 변형 | 동작 |
|---|---|
| 끔 | 수동 조작만 |
| 회전 | OrbitControls autoRotate (느린 자동 회전) |
| 투어 | **focus 노드 순회** 시네마틱 줌(3.2s 간격). focus 없으면 허브 상위 6개 |

## 6. 성능 (PERF-ARCH 예산)
- 프레임 애니메이터(`animateNodes`)는 **상태뷰·idle·호버·인트로·펄스 중 하나라도 활성일 때만** O(n) 순회, 그 외 비용 0.
- 새 지오메트리 생성 없음(전부 기존 메시의 scale/emissive 변조) → 수천 노드 확장 시에도 가벼움.
- 확장 시 가드: 동시 렌더 수는 레이어/상태 필터 + LOD로 상한(PERF-ARCH §3).
